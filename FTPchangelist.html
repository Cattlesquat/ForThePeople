For The People VASSAL Module Change List

3.5 Changelist

Improvements

  • Updated custom code to prevent "Obsolete Custom Code" messages with latest Vassal.
  • Changed module settings to allow it to take advantage of improved Vassal performance
  • Union Control of Mississippi now correctly adds 10 to USA Strategic Will (rather than subtracting 10 from Confederate). Thx Gabriel Schweitzer for bug report.

3.4 Changelist

Improvements

  • Corrected a problem where pieces dragged together would sometimes "unstack".

3.3 Changelist

Improvements

  • Can now place 8 Union SP in Washington DC all at once (with Alt+8 or right-click menu)
  • Takes advantage of various nice VASSAL 3.4.0 features
  • There is now a much more noticeable "drag threshold" (10 pixels instead of 1), so pieces/stacks won't be so pixel-sensitive and fidgety.
  • Rollover text for groups of units does a better job of picking up other counters underneath (e.g. trenches) as well
  • "Change Sides" button now defaults to opposite site for quicker "Hot Seat" play
  • Ctrl+Mousewheel will zoom the map in and out (YAY! They let me put this in VASSAL! Coming soon to ALL your games...)
  • Mousewheel in general should "scroll text windows faster", it used to be infuriatingly slow

Bug Fixes

  • When an army was eliminated in the same space as an opposing army, extra SW penalty was being charged. This now correctly happens ONLY when an army is eliminated in the same space as another *friendly* army.
  • Ewell and AP Hill were incorrectly missing from some scenario maps - fixed

3.2 Changelist

Improvements

  • NOTE: any time the version number changes, I do NOT recommend using saves made with earlier versions of the module -- finish any current games using the version you started it with. Although they will "kind of work", you will not actually most of the new improvements, especially the performance-related ones.
  • Auto Organize Stacks now defaults to "off" - apparently my computer is faster than I thought it was :)
  • Substantial performance improvements, particularly with auto-organize stacks off.
  • Armies now have Ctrl+P hotkey to "Convert/Place PC Marker"
  • Strength Point counters now have Expand (Ctrl+X) and Consolidate (Ctrl+C) commands -- e.g. a 6 turns into a pile of six ones, and vice versa.
  • Forts can be built by selecting empty spaces and picking Ctrl+F (CSA) or Alt+F (USA). So that you don't have to go drag one all the way across the map from the fort pool.
  • "Middle" auto-organize level is now much faster but pared down.
  • Game log reporting now allows bolding, multiple colors, etc, for better clarity.
  • Dice display symbols in log when rolled -- so cute!
  • Scrolling improved in "How to Play" section.
  • General cleanup and polish throughout.

Bug Fixes

  • Little Rock was missing from resource site list; fixed.
  • Emancipation event was missing its card text.
  • Create-Army options were sometimes greyed out improperly for generals.
  • Some cards when played for the event weren't displaying their card text.
  • Cavalry brigades eliminated from army displays weren't always updating the Army SP count.
  • Various typos fixed in card text, etc.

3.1 Changelist

Improvements

  • Armies now display commander's Strategy rating on left side of counter.
  • Army counter contains "*" if Army has at least one Cav Brigade, or "!" if it is a MONGOL ARMY of only Cavalry(!)
  • When randomizing generals on a map or board location (e.g. for casualties rather than for initial placement), the name of the top General in the resulting stack is reported in the log, to improve clarity that the top one in the resulting stack is the intended randomized casualty.

Bug Fixes

  • Maintenance key added to force initial PC markers to self-register.
  • State PC status now properly disappears when a border state flipped from CSA to USA control.
  • Army's internal cavalry count now updated properly when a Cav Brigade is flipped to 2 SP. (was previously okay if 2 SP cav moved in or out, but not if flipped right there inside army.

3.0 Changelist

Improvements

  • All-new module!
  • Latest hi-res art supplied by Mark Simonitch from GMT.
  • Much automation now available to make record-keeping easier.
  • Automatic stack reorganization
  • Wherever possible, on-map charts have been turned "right side up" for ease of on-screen play.
  • Java source for custom classes incorporated into module.
